# Session: Daikin WiFi v0.1.1 Hotfix

**Date:** 2026-05-18  
**Tag:** daikin-v011-hotfix  
**Commit:** c28882f

## Two-Bug Pattern

Mads installed v0.1.0 and hit two runtime errors immediately:

### Bug 1: Groovy Write-Only Property Shadowing

Custom command `setSchedule(schedule)` (line 312, Thermostat capability stub) follows JavaBean conventions, creating a write-only property named `schedule`. When `registerSchedules()` called `schedule(cron, method)` (line 685), Groovy's dynamic dispatch resolved the name as the write-only property (read-only, no getter) instead of the platform's `schedule()` method.

**Fix:** Rewrote `registerSchedules()` to use idiomatic `runEvery1Minute()`, `runEvery5Minutes()`, etc. — zero naming conflicts.

### Bug 2: Non-Existent HubAction Constructor Overload

Driver used `new HubAction(Map params, Protocol protocol, Map options)` — a 3-argument form that does not exist in current Hubitat firmware. The `callback` key was in a separate third Map.

**Fix:** Moved callback into params Map, using documented 2-arg form: `HubAction(Map params, Protocol protocol)`.

## Lesson: Test on Install

Both bugs were immediately visible on first `Save Preferences` — fast feedback loop. Smoke test on hub before tagging would catch these proactively.
