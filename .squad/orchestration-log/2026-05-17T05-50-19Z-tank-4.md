## Scope

Fixed `.github/workflows/release.yml` changelog discovery for multi-file drivers (parent/child pairs).

## Problem

Changelog discovery looked for `${slug}.groovy` (not found for SunStat parent because file is `sunstat-thermostat-parent.groovy`). Fell back to alphabetically-first `*.groovy`, which is often the child file. Result: "No changelog entry found" error on SunStat v0.1.4 release, aborting the workflow.

## Solution

Implemented three-tier discovery:
1. Primary: `${slug}.groovy` (unchanged, for single-file drivers)
2. Secondary: `*-parent.groovy` when present (new, for multi-file drivers)
3. Tertiary: alphabetical fallback (preserved for drivers without parent/child)
