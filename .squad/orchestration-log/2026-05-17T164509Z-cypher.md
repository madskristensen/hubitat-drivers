## Task

Research consumer-auth alternatives for Bosch Home Connect driver, triggered by user directive to eliminate developer portal onboarding step.

## Work Completed

1. **Surveyed 5 candidates:**
   - hcpy / hcpy2-0 (local WebSocket + PKCE against SingleKey ID)
   - Home Assistant `home_connect_alt` (developer API)
   - openHAB Home Connect Direct Binding (desktop GUI profile downloader)
   - Homebridge plugins (all require developer API)
   - Direct SingleKey ID consumer SSO (form-scraping / PKCE)

2. **Key findings:**
   - CAPTCHA added to SingleKey ID (circa 2024) blocks scripted auth
   - hcpy operational protocol is local WebSocket (TLS-PSK, CHACHA20-POLY1305 cipher)
   - Hubitat sandbox lacks WebSocket client + non-standard TLS cipher support
   - Consumer backend (`eu.services.home-connect.com`) has no REST polling API for state
   - Developer API (`api.home-connect.com/api/`) provides proper cloud-mediated REST interface

3. **Decision:** No viable consumer-auth-only path for Hubitat. Developer portal registration remains unavoidable.

4. **Recommendation:** Proceed with Option 1 (Developer App, Device Flow). One-time 5-minute onboarding is worth the UX clarity.
