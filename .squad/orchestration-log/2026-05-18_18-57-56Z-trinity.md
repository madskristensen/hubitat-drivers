## Mission

Capability-gap analysis of `eriktack/hubitat-daikin-wifi` upstream driver. Assess fix scope + priority ranking.

## Findings

### Critical Missing Capabilities
1. **No `supportedThermostatModes` declaration** — breaks Rule Machine thermostat mode dropdowns on any integration
2. **No `initialize()` lifecycle** — polling dies on hub restart (no re-registration)
3. **Energy endpoint over-polling** — 1440x per day at 1-min refresh; should be ~48x on 30-min schedule

### Priority Fix List (8 items, ordered)
1. Guard `otemp`/`htemp` sentinel values (`"-"` → `NumberFormatException`)
2. Add `supportedThermostatModes` emission in `installed()` + `updated()`
3. Fix `supportedThermostatFanModes` (missing in `installed()`)
4. Add `initialize()` lifecycle chained from `installed()` + `updated()`
5. Throttle energy polling to separate 30-min schedule
6. Add HealthCheck capability + `lastActivity` (LAN driver pattern A)
7. Add econo/powerful mode support
8. Apply event hygiene (`descriptionText` + `emitIfChanged()`)

### Scope Estimates
- **Items 1–5** (trustworthy, Rule Machine-compatible): ~4–5 hours
- **Items 1–8** (repo-quality, shelf-stable): ~16–20 hours
