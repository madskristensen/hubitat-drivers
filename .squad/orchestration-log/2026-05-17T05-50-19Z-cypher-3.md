## Key Findings

**setColor 400 — three candidate causes (highest to lowest confidence):**
1. `pattern.id = generatePatternId()` produces non-UUID string `"hubitat-{ts}-{n}"` instead of UUID format. Gemstone API validates pattern IDs as UUIDs.
2. `hubitatHueSatToArgb` + `kelvinToArgb` produce `0x00RRGGBB` (alpha byte = 0). API expects `0xFFRRGGBB` (fully opaque).
3. `referencePatternId = null` (explicit null) should be absent instead.
