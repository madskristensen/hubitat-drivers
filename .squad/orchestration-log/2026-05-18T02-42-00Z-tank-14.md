## Lessons Captured

1. **Dependency cost is visible only after download** — Gradle doesn't announce 150MB downloads upfront; it's a discovery during setup. Pre-flight validation should have surfaced this trade-off earlier.
2. **Driver repo ≠ software project repo** — This repo's users expect to clone, pick a `.groovy` file, and install in Hubitat. Anything that changes that model (build systems, Java deps) is out of scope.
3. **Hardware validation > unit tests** — For embedded device drivers, real hardware testing (Switch's role) catches integration bugs that unit tests cannot.
