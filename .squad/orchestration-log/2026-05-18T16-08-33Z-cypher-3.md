- **Sandbox:** All safe — no reflection, no native libraries, no MQTT

### Protocol — Bosch Home Connect

- **Auth grant types supported:** Authorization Code (used by craigde) + Device Flow (alternative, not used)
- **Access token lifetime:** 24 hours
- **Token refresh limits:** 100/day, 10/minute
- **Daily request limit:** 1,000 per client+user
- **Rate limit per minute:** 50 (burst 20, leaky bucket)
- **Door-state endpoint:** GET /api/homeappliances/{haId}/status → door states map to ContactSensor
- **Door-alarm events:** SSE stream → PushableButton integration
- **Rate limit implications:** REST-only polling at 120s intervals = ~720 calls/day (within budget); SSE eliminates polling overhead

### OAuth in Hubitat — Authorization Code Grant

**Pattern discovered:** Hubitat App + `mappings {}` + `getFullApiServerUrl()` cloud callback endpoint for OAuth2 redirect URI.

- **Hubitat callback URL:** `https://cloud.hubitat.com/api/{hub-uuid}/apps/{app-id}/oauth/callback`
- **Hub UUID:** Stable per-hub, so per-hub registration required in Bosch portal (one-time setup per user)
- **Token flow:** `httpPost` to Bosch token endpoint → access+refresh tokens stored in App's `atomicState` (survives reboots)
- **Token refresh:** Proactive, request-gated via `getOAuthToken()` before each API call
- **Status:** Works reliably (confirmed across multiple cloud-OAuth drivers in community: SunStat, Gemstone, Daikin)

**This pattern unblocks** future Hubitat drivers that require OAuth2 cloud authentication (Google Home, Amazon Alexa, etc.).
