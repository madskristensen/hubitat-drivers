## Bug Description

**Live symptom (reported by Mads):**
```
No signature of method: hubitat.helper.interfaces.Mqtt.connect() is applicable for argument types: 
(java.lang.String, GStringImpl, null, null, java.util.LinkedHashMap)
```

**Root cause:** v0.4.0's `mqttConnect()` method called:
```groovy
interfaces.mqtt.connect(brokerUrl, clientId, username, password, optionsMap)
```

This 5-arg form with a Map for LWT (Last Will & Testament) options is NOT supported by Hubitat's Mqtt interface.

---

## Solution

**Choice:** Option A — 8-arg form (Hubitat documented API)

Switched to the documented 8-arg method:
```groovy
interfaces.mqtt.connect(
  brokerUrl,           // String
  clientId,            // String
  username,            // String
  password,            // String
  lwtTopic,            // String - fully/hubitat/state
  lwtQos,              // Integer - 1
  lwtRetained,         // Boolean - true
  lwtPayload           // String - "offline"
)
```

**LWT behavior preserved:** On disconnect, publishes `"offline"` to `{prefix}/hubitat/state` with QoS 1 and retained flag. On successful connect, `mqttClientStatus()` publishes `"online"` to the same topic.
