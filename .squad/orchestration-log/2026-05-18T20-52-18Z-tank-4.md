**Task:** Daikin v0.1.2 architecture upgrade

## Trigger

v0.1.1 HubAction(Map, Protocol) 2-arg constructor failed on Mads's Hubitat firmware (after v0.1.1 had swapped from the 3-arg form). Both 2-arg and 3-arg Map-based HubAction overloads proved unreliable across firmware versions.

## Scope

Replace HubAction entirely with asynchttpGet (Hubitat's documented modern HTTP-over-LAN API).

## Outcome

**Commit:** e45967e on main

**Changes:**
- sendGet() now uses asynchttpGet() with full http://ip/path URI
- All 6 callback handlers rewritten to AsyncResponse signature
- Zero HubAction/HubResponse/sendHubCommand/Protocol.LAN references in executable code
- Version 0.1.2 in all 4 places (header, DRIVER_VERSION, changelog, packageManifest.json)

**Note:** Mads reported v0.1.1 actually worked for him after a re-install (he was running the latest before Tank-4 finished). v0.1.2 is therefore an architecture upgrade rather than a strict bug fix — same behavior, but version-stable across firmware variants.
