## Design Notes

**cycleEffect() flow:** User pressing "next effect" in UI still works correctly. Edge case: if the effect catalog contains only one effect, advancing to "next" wraps back to the same effect and the skip-if-match skips the PUT (acceptable — no user-initiated animation restart, consistent with idempotency goal).
