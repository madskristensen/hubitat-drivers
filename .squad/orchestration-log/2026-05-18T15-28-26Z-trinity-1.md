### Rubric Structure

**7 Weighted Criteria (max 100 pts):**
1. **Local vs. Cloud Protocol** (20 pts) — Local LAN 20, cloud REST 10, sketchy cloud 5, cloud-killed 0
2. **Mads Can Test** (15 pts) — Owned <$100 (15), costs $100–$500 (7), >$500 (0)
3. **User Demand Signal** (15 pts) — 2+ forum threads (15), 1 thread (10), Mads noticed (5), solo idea (0)
4. **Sandbox-Safe** (15 pts) — Pure Groovy (15), parent/child workaround (10), custom feasible (5), JNI/reflection (0)
5. **Vendor API Stability** (15 pts) — Local vendor stable >3y (15), cloud stable (10), fragile (5), killed (0)
6. **Effort to Ship** (10 pts) — Single-device local (10), parent/child cloud (5), >80h/new pattern (0)
7. **Maintenance Burden** (10 pts) — Local + vendor docs (10), cloud/reverse-eng (5), killed/complex (0)

**Thresholds:**
- 80–100 pts: ✅ Strong Fit — prioritize
- 65–79 pts: 🟡 Conditional Fit — check hard disqualifiers
- 50–64 pts: ❌ Weak Fit — defer
- <50 pts: 🔴 No Fit — skip

### Hard Disqualifiers (Any ONE = OUT, regardless of score)

1. Cloud API officially dead or hostile (MyQ post-Oct-2023, C&D issued)
2. Requires reflection, JNI, or external native libs
3. Device >$500 or unavailable
4. Browser OAuth2 redirect required
5. Persistent MQTT subscriber needed
6. Binary protocol with no Groovy decoder
7. Safety-critical without `log.info` audit trails (garage, lock, gate)
8. >1KB secrets in preferences
9. Multi-protocol undocumented fallback
10. Uses `getClass()`, reflection, or sandbox-restricted Groovy

### Cloud-Service Trigger Patterns Documented

**Pattern A:** Cloud-polling parent + children (e.g., SunStat)  
**Pattern B:** Cloud-polling single device, no parent/child (e.g., Gemstone)  
**Pattern C:** Webhook relay (cloud → Maker API → driver event)  
**Pattern D:** Hybrid polling + webhook (backup resilience)

Each pattern documented with use cases, code reuse, testing requirements, and antipatterns.

### House Style (Part 1)

**Device classes embraced:** Thermostats (Daikin local), lighting (Gemstone cloud OAuth), fireplaces (Touchstone Tuya), actuators  
**Non-negotiable patterns:**
- Local-first if available
- Single protocol per driver (no complex fallbacks)
- Parent/child for multi-device clouds
- Clean Hubitat citizen (7-point audit pass)
- Mads owns or will buy for testing
- Graceful degradation (cloud breaks, local survives)

**What we skip:** Multi-protocol, paid-gateway integrations, reflection, large dependencies, no audit logging
