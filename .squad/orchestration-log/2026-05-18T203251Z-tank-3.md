# Tank-3 — Daikin WiFi v0.1.1 Hotfix

**Date:** 2026-05-18T20:32:51Z  
**Status:** ✅ SHIPPED  
**Commit:** c28882f

## Summary

Tank diagnosed and fixed two runtime errors hit on v0.1.0 install:

1. **Schedule property shadowing** (line 685): `setSchedule()` custom command created a write-only property that shadowed the Hubitat `schedule()` method. Fixed by rewriting `registerSchedules()` to use `runEvery*()` idiomatic methods.

2. **Invalid HubAction constructor** (lines 413–418): Driver used non-existent 3-arg `HubAction(Map, Protocol, Map)` form. Fixed by moving callback into params Map and using documented 2-arg `HubAction(Map, Protocol)` form.

Version: 0.1.0 → 0.1.1 (patch)

## Lessons

1. Thermostat capability drivers MUST NOT call `schedule(cron, method)` — the `setSchedule()` stub creates a shadowing property. Use `runEvery*()` instead.
2. HubAction 3-arg Map-Map-Map form does not exist in current firmware. Callback belongs in params Map.
3. Test on first install before shipping — both bugs were immediately visible.

## Trigger

Mads installed v0.1.0 and hit runtime errors on first `Save Preferences`.
