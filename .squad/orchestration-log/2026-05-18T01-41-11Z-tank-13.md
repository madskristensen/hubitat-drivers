**Key architectural decisions:**
- controllerName stored as simple text preference (not UUID or enum). Users provide the controller name from the Gemstone app or device page.
- handleDevicesResponse() performs case-insensitive match: `devices.find { it.name?.toLowerCase() == controllerName.trim().toLowerCase() }`.
- If no match: `log.warn` with available names list, then fall back to `devices[0]` (driver continues operating).
- availableControllers stored in `state` (diagnostic only, not a device attribute), populated after successful discovery.
- Backward compatibility: blank/null controllerName → pick first device + multi-controller warning (if size > 1). When controllerName is set, warning suppressed.
- No driver architecture changes; Hubitat handles multiple device instances natively.
- Each device independently authenticates (shared Cognito client ID, separate token cache) and polls; no inter-device coordination needed.

**Status:** Ready for Switch hardware validation (multiple controllers confirmed, binding works, graceful fallback verified).
