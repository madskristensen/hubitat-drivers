## Ship 3: Touchstone v0.1.20 — Active-TCP IP Discovery (DHCP-Renewal Recovery)

**Commit:** ffbfd08  
**Files Modified:** `drivers/touchstone-fireplace/touchstone-fireplace.groovy`, `TESTING.md`, `packageManifest.json`

**What Changed:**

**3A: Improved Error UX**
- Changed `openSocket()` failure from `log.warn` to `log.error` with actionable text
- States IP that was tried
- Recommends DHCP reservation + suggests pressing Discover
- Guidance to update preferences if needed

**3B: Active-TCP Discovery State Machine**
- New command: `discover` (zero-arg button on device page)
- New attribute: `networkAddress` (string) — surfaces discovered IP

**Discovery Flow:**
```
discover()
  → build state.discoveryProbeQueue (smart ±20 range first, then 1-254 sweep)
  → closeSocket + unschedule heartbeat
  → state.discoveryMode = true
  → runIn(1, "discoveryProbeNext")

discoveryProbeNext()
  → if queue empty → discoveryComplete()
  → pop next octet, rawSocket.connect(targetIp, 6668)
  → send TUYA_CMD_DP_QUERY frame
  → runIn(3, "discoveryProbeTimeout")

socketStatus() [modified]
  → if discoveryMode + error: cancel timeout, probeNext
  → if discoveryMode + other: debug log only

parse() → processFrame() [modified]
  → if discoveryMode: route to discoveryHandleResponse(response)

discoveryHandleResponse()
  → if devId matches stored deviceId: update deviceIP pref, emit networkAddress, discoveryComplete()
  → if devId mismatch: debug log, probeNext

discoveryProbeTimeout()
  → if discoveryMode: probeNext

discoveryComplete()
  → state.discoveryMode = false
  → initialize() [restores normal socket + heartbeat]
```

**Guards Added:**
- `openSocket()`: skip if discoveryMode
- `reconnectSocket()`: skip if discoveryMode
- `sendHeartbeat()`: skip if discoveryMode
- `parse()`: skip normal post-processing if discoveryMode

**Known Limitations (Switch to Verify):**
- gwId in v3.3 DP_QUERY response must be confirmed on real hardware
- Hub sandbox may rate-limit rapid sequential rawSocket.connect()
- Scan time: 2s/IP worst case → ~8 min full sweep; smart ±20 typically <1 min
- IP outside /24 requires manual entry (set DHCP reservation to avoid)

**Design Decisions:**
- Fail-closed on devId: only accept if `response.devId == storedDevId`
- Pre-computed probe queue in state (simpler than phase flags)
- `intentionalCloseAt` reuse (avoids new socket-suppression logic)
- `discoveryComplete()` always calls `initialize()` (clean state handoff)
