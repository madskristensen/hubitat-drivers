## Release v0.3.0: Custom Named-Effect Control

**User Request:** Mads needs to call effects by human-readable name from Rule Machine, not by opaque UUID.

**Decision:** Implement a named-effect cache (`state.effectCatalog` as `name → patternId`, `state.effectPatterns` for full payloads) with async refresh and 1-hour TTL.

**API Endpoints (from Cypher's spec):**
- `GET /folders/pattern/list?page=N` — user presets
- `GET /downloads/folders/pattern/listGemstoneManaged?page=N` — Gemstone built-ins

**Implementation:**
- Added `refreshEffectCatalog()` helper to fetch and merge both sources into case-insensitive lookup table.
- User presets beat built-ins on collision.
- New command: `setEffect(String name)` resolves name → patternId and activates via `PUT /deviceControl/play/pattern`.
- On-demand catalog refresh: if cache is stale or name unknown, async refresh is triggered.
- Logged available effects on each refresh for discoverability.

**New Attributes:**
- `effectName` — human-readable active effect

**Skills Created/Updated:**
- Added `state.effectCatalog`, `state.effectPatterns`, `state.effectCatalogFetchedAt` to `.squad/skills/`.

## Release v0.4.0: Standard Hubitat Capabilities + Favorites Priority

**User Requests:**
1. Integrate with Hubitat's standard `LightEffects` capability (dashboard dropdown support).
2. Make user-marked favorites appear **FIRST** and be visually distinct.
3. Add `ColorTemperature` support (Kelvin-based) as a separate mode from effects.

**Directive:** Mads emphasized that favorites are the primary daily-use surface; they must not be buried in a 100+ pattern catalog.

**Decisions Implemented:**

### 1. LightEffects Capability
- Standard `setEffect(BigDecimal index)`, `setNextEffect()`, `setPreviousEffect()` commands.
- `lightEffects` JSON attribute as a favorites-first map with numeric string keys (`"0"`, `"1"`, ...).
- Index `0` is the **first favorite**, not an `Off` placeholder.
- Preserved existing `setEffect(String name)` overload for Rule Machine backward compatibility.

### 2. Favorites Separation
- `state.favorites` stores ordered favorites in Gemstone's returned order.
- `state.effectCatalog` still holds the full catalog (for complete discoverability).
- `favoriteEffects` attribute exposes comma-separated favorite names.
- Discovered via `isFavorite` boolean in pattern records (Cypher's spec).
- Star (⭐) prefix applied only on user-facing output, NOT in raw lookup maps.

### 3. ColorTemperature Fallback
- Cypher's spec documents no native Kelvin/CCT endpoint.
- Implemented as RGB white-spectrum conversion: `setColorTemperature(kelvin, level?, transitionTime?)`.
- Marks `colorMode = CT` explicitly for mode tracking.
- Still sends RGB wire payload via `PUT /deviceControl/play/pattern`.

### 4. Mode Tracking
- `colorMode` attribute tracks active mode: `RGB`, `CT`, or `EFFECTS`.
- `refresh()` re-infers mode from pattern heuristics.

**Implementation Impact:**
- `refreshEffectCatalog()` now rebuilds `state.favorites`, `state.effectCatalog`, `state.effectIndex`, `lightEffects`, and `favoriteEffects` together.
- Switching from effect to solid color generates a fresh pattern ID to avoid misclassification on next refresh.
- Runtime favorite/total counts logged on every catalog refresh.

**New Attributes:**
- `lightEffects` — JSON map, favorites-first
- `favoriteEffects` — comma-separated names
- `colorMode` — `RGB`, `CT`, or `EFFECTS`
