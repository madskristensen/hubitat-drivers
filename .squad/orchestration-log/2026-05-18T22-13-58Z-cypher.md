## Work Done

1. **Official MyQ API Status Investigation**
   - Verified Chamberlain CTO statement (October 25, 2023) blocking third-party API access
   - Confirmed no public developer program exists for individuals or open-source projects
   - Reviewed authorized partners list: no Hubitat, no open-source platforms

2. **Community Cloud Ecosystem Review**
   - `pymyq`: Reverse-engineered endpoints; likely non-functional as of 2026
   - Home Assistant integration: Removed December 2023
   - `homebridge-myq`: Retired; author migrated to ratgdo
   - Verdict: No viable cloud workaround

3. **Local Hardware Alternatives Assessment**
   - **ratgdo (ESPHome firmware):** Active maintenance, latest release April 25, 2026; stable documented REST API on port 80
   - **Konnected GDO blaQ:** Protocol-identical to ratgdo; explicitly advertises Hubitat support; $89
   - **MQTT firmware (legacy):** Feasible but Hubitat has no built-in MQTT client
   - **Shelly relays:** Dry-contact only; doesn't support Security+ 2.0

4. **Risk Register Development**
   - Entity name drift, API schema changes, project abandonment, IP address changes
   - Mitigation strategies documented
   - Polling latency (5s) accepted as design constraint

---

## Recommendation

**Build ratgdo ESPHome HTTP driver** using `asynchttpGet` polling.

- Local REST API on port 80: fully documented, actively maintained
- Covers both ratgdo and Konnected blaQ hardware
- Capabilities: `GarageDoorControl`, `ContactSensor`, `Switch` (light), `HealthCheck`
- Poll cycle: 5s default interval
- No cloud dependency, no auth overhead by default
- Matches this repo's local-first policy
